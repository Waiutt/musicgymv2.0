## 一、音乐模块 🎵 — 打造真正的"运动音乐"体验

### v2.5 目标：从"能播"到"好用"

| 序号 | 功能 | 说明 | 工作量 |
|------|------|------|--------|
| 1.1 | **Spotify SDK 集成** | 接入官方 Android SDK，用户登录后读取 Spotify 歌单、搜索歌曲、完整播放（需 Premium） | 3h |
| 1.2 | **音频特征获取** | Spotify Audio Features API → 获取每首歌的 BPM、能量值(energy)、舞动性(danceability) | 1h |
| 1.3 | **均衡器** | 5 种运动预设（跑步/举铁/拉伸/人声/自定义）+ 5-16 频段手动调节。Android 原生 Equalizer API | 2h |
| 1.4 | **歌词显示** | LRC 格式解析 + Musixmatch API + 逐行高亮滚动。`Handler` 同步时间轴 | 2h |
| 1.5 | **歌单管理** | Room 新增 playlists 表 + playlist_songs 关联表。创建/删除/重命名/导入 Spotify 歌单 | 2h |
| 1.6 | **红心收藏** | 播放器加 ❤ 按钮 + 自动归入"我喜欢"默认歌单 | 0.5h |
| 1.7 | **无缝切换** | Gapless playback + 交叉淡入淡出(Crossfade)。`MediaPlayer.setNextMediaPlayer()` | 1h |

### v2.6 目标：步频 BPM 匹配（核心差异化功能）

| 序号 | 功能 | 说明 | 工作量 |
|------|------|------|--------|
| 2.1 | **步频检测** | `Sensor.TYPE_ACCELEROMETER` + 峰值检测算法。实时计算步频(spm) | 3h |
| 2.2 | **BPM 匹配引擎** | 步频 ±5% 内优先匹配 + 能量值加权排序 + 自动切歌 | 2h |
| 2.3 | **运动模式自动切换** | 跑步→高BPM·高能量 / 举铁→低BPM·高能量 / 休息→低BPM·低能量 | 1h |
| 2.4 | **节奏引导变速** | 音乐 BPM 提前 5% 渐进式引导用户加快步频 | 2h |

> **最终形态**：用户跑步时不用碰手机，音乐自动根据步频切换。这就是 Strava、Keep、网易云都不做的功能。

## 二、力量训练模块 🏋️ — 从"能用"到"最好用"

### v2.7 目标：可视化和智能化

| 序号 | 功能 | 说明 | 工作量 |
|------|------|------|--------|
| 3.1 | **动作演示 GIF** | 每个动作配 GIF 动图演示。打包在 assets/ 或从 Firebase Storage 加载 | 4h |
| 3.2 | **训练计划生成** | 基于用户目标（增肌/减脂/力量）和历史数据，自动生成 4 周训练计划。接入大模型 API | 4h |
| 3.3 | **肌肉恢复状态** | 记录每个肌群上次训练时间 → 计算恢复百分比 → "胸部: 72h 前训练, 可以再练" | 2h |
| 3.4 | **渐进超负荷建议** | 上次卧推 80kg×8 → 本次建议 82.5kg×8。`PR × (1 + target%)` 自动计算 | 1h |
| 3.5 | **热身组标记** | 每动作支持标记热身组（不计入容量统计） | 1h |
| 3.6 | **超级组支持** | 两个动作绑定为一组，轮流执行，共享休息计时 | 2h |
| 3.7 | **训练笔记** | 每动作/每组可添加文字备注（"状态不好"、"下次加重"） | 1h |
| 3.8 | **RPE 自感用力评分** | 每组训练后输入 RPE 1-10，长期追踪训练强度趋势 | 1h |

## 三、数据统计模块 📊 — 从"展示"到"洞察"

### v2.8 目标：职业级分析

| 序号 | 功能 | 说明 | 工作量 |
|------|------|------|--------|

| 4.5 | **体重趋势图** | ProfileFragment 里的体重记录从文字列表→折线图 | 2h |

| 4.7 | **周报推送** | 每周一自动生成上周运动总结：跑了几天/总距离/力量训练次数。通知栏推送 | 2h |
